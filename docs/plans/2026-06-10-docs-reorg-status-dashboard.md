# Docs reorg — plan status legibility + STATUS dashboard

## Problem

The repo's docs taxonomy is sound, but its *status* signal has rotted: the
plans index drifted from frontmatter reality (wrong rows, missing rows,
non-schema statuses), several plans marked `in-progress` shipped weeks ago,
GPU-dependent plans stayed `planned` after the 4090s were sold (2026-05-16),
and there is no single place to answer "what's the current state of the lab
and where is it headed". Separately, the canonical quick-reference for hosts
and the network lives outside the repo (`~/.claude/HOMELAB.md` on the
operator's machine) and contradicts repo docs in places.

## Plan

### Phase A — plan status hygiene + generated index (this PR)

- Audit all plan frontmatter against git history and manifests; fix statuses.
- Add a `summary:` field to every plan so the index is fully generated.
- Add optional `superseded_by:` and `blocked_on:` frontmatter fields.
- Replace the hand-maintained Document Index in `docs/plans/README.md` with a
  status-grouped block generated by `scripts/plans-index` (Go, stdlib only)
  between `BEGIN/END PLANS INDEX` markers.
- Wire `plans-index -check` into CI so index drift and schema violations fail
  the build.

### Phase B — content migration + STATUS dashboard (follow-up PR)

- Merge `~/.claude/HOMELAB.md` content into the repo: VLAN/IP ranges and IPMI
  into `docs/architecture/networking/`, hestia hardware/NIC history and the
  service table into `hosts/hestia/README.md`, decommissioned GPU stack notes
  into `hosts/hestia/llms/README.md`, hermes backend state into
  `docs/operations/apps/hermes.md`, memory-bench runbook fixes into
  `hosts/hestia/bench/memory/README.md`.
- New `docs/STATUS.md`: cluster/hosts at a glance, in-flight plans, next up,
  recently completed, known issues. Update convention: touched by any PR that
  changes a plan status, lands an incident, or changes hardware/topology.
- Fix stale claims (single-node cluster references, "GPU server" descriptions).
- Pointers from root `README.md`, `AGENTS.md`, `docs/README.md`.

### Phase C — operator-side cleanup (no PR)

- Shrink `~/.claude/HOMELAB.md` to a pointer at `docs/STATUS.md` plus
  access-only quirks (SSH user, sudo posture, `noexec /home`, IPMI).

## Verification

- `make plans-index-check` passes; CI job green.
- Every plan file carries schema-valid frontmatter.
- `docs/STATUS.md` answers "current state / in flight / next" without opening
  any other file.
